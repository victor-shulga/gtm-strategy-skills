# 07 · Offers

Сьомий крок. Перетворити ринок/ICP/тіри (03) + VP (06) на **конкретні офери, які можна продавати й
тестувати**. Cold audit → офери це гіпотези; proof/ціни — діапазони з позначкою «валідувати». Без гейту.

Вхід — 03 (ринок/тіри/персона), 06 (dream outcome, pains, benefits, why-now), 05 (competitor whitespace),
04 (sizing — для sanity цін).
Прочитай `${CLAUDE_PLUGIN_ROOT}/reference/agency-stages.md`.

## Реюз
`offer-factory` (генерація offer-bets ICP×pain×mechanism) · `offer-ladder` (вертикальна драбина) ·
`offer-definer` (features→outcome). `anticopywriting-ai` — щоб назви/копі не пахли ШІ.

## Що зібрати

**1 · Offer ladder (вертикальна драбина навколо ОДНІЄЇ трансформації).**
- *Hook / free* — низькобар'єрний вхід (аудит/розрахунок втрат) = sales-opener + (на 2+) лід-магніт.
- *Entry (paid pilot)* — швидкий платний пілот, що доводить ROI (фіксований скоуп/ціна/строк).
- *Core* — основний оффер (retainer/build) після доведеного пілота.
- *Continuity* — managed/оптимізація щомісяця (LTV).

**2 · Offer-картки (Grand Slam) — на core + entry.** Поля:
- *Name* (зрозуміла, outcome-орієнтована, не «AI consulting»).
- *For (ICP/тір) · Trigger · Pain* (з 03/05).
- *Dream outcome* (big promise з 05).
- *Mechanism / what's included* (скоуп: що саме робимо, у їхньому стеку).
- *Measurable outcome* (метрика: +%/+N/$; де нема — «proof TBD з пілота»).
- *Speed* (час до результату) · *Effort removed* (done-for-you / no new tool).
- *Proof* (кейс/запис/гарантія; TBD на Стадії 0).
- *Price / structure* (діапазон + модель: фікс пілот → retainer).
- *Risk reversal / guarantee* (ROI-gated, refund/extend — знижує бар'єр на Стадії 0).
- *CTA* (наступний крок).

**3 · Use case (1 на core, шаблон Z1):** Persona / Trigger / Pain / Solution / Outcome / Proof.

**4 · Validation plan (Z1-T19):** показати картки 5-10 людям (мережа + cold), 8 питань, зібрати
sentiment + реакцію на ціну; ціль ≥70% «розглянули б» і ≥1-2 у реальні пілоти.

## Stage-adaptive глибина
- **Стадія 0-1:** **1 core + 1 entry** + hook; проста драбина; 1 use case; ціни-діапазони. Без портфеля.
- **Стадія 2:** продуктизувати 2-3 офери + тіри ціни; use case library; light A/B framing.
- **Стадія 3-4:** повний портфель + outcome-based pricing + upsell-драбина.

## Принципи
- Офер = **persona × trigger × pain × mechanism × outcome × proof**, не «послуга» (не «Web Dev», а
  «30-day Shopify CRO для DTC, що втрачає конверсію»).
- Підсилюй цінність не зниженням ціни, а **proof + guarantee + speed + low-effort** (Grand Slam).
- Entry-оффер мусить **сам доводити ROI** (на Стадії 0 це двигун перших кейсів).

## Вихід (Notion)
Під-сторінка **«06 · Offers»**: offer ladder (таблиця) · 1-2 offer-картки · use case · validation plan ·
блок «Джерела / припущення». Українською; назви оферів — мовою ринку (часто EN).

## Definition of Done
- Offer ladder (hook → entry → core → continuity) визначено.
- ≥1 core + ≥1 entry offer-картки за повним шаблоном Grand Slam (з risk-reversal).
- 1 use case (шаблон Z1) + validation plan (5-10 розмов, критерій ≥70%).
- Ціни — діапазони з позначкою «валідувати»; proof «TBD» де нема. Notion-сторінка створена.
